# Contributing to SchoolDesk

Thanks for your interest in contributing to SchoolDesk! We welcome all contributions, from bug fixes and documentation to new features.

## How to Contribute

1. **Fork the Repository**: Create your own fork of the project.
2. **Setup Local Environment**: Follow the instructions in the [README](README.md).
3. **Create a Branch**: Use a descriptive branch name (e.g., `feature/attendance-reports` or `fix/sync-error`).
4. **Make Your Changes**: Ensure your code follows the existing style and is well-documented.
5. **Add Tests**: If you're adding a new feature, please include relevant tests.
6. **Submit a Pull Request**: Provide a clear description of what your PR does and why it's needed.

## Development Workflow

- We use **Prettier** for code formatting.
- Ensure all TypeScript types are correctly defined.
- Run `npm run dev` to test your changes in the Electron environment.
- Use `npm run setup:appwrite` if you make changes to the database schema.

## Community

If you have questions or need help, please open an issue on GitHub.
